# API Capture & Analyzer 浏览器扩展

一个强大的浏览器扩展，用于自动捕获和分析网页中的所有网络请求，并智能标注接口类型。

## 功能特性

### 🎯 核心功能
- **自动捕获**: 实时捕获页面中的所有网络请求（XHR、Fetch、普通请求）
- **智能标注**: 自动识别接口类型（RESTful、数据接口、文件接口等）
- **详细分析**: 显示请求方法、URL、状态码、耗时等详细信息
- **实时统计**: 显示总请求数、独立域名数、API接口数

### 🔍 接口类型识别
- **RESTful API**: `/api/`, `/rest/`, `/graphql` 等
- **数据接口**: `/data/`, `/json/`, `/xml/` 等
- **文件接口**: `/upload/`, `/download/`, `/file/` 等
- **统计分析**: `/analytics/`, `/tracking/`, `/metrics` 等
- **搜索接口**: `/search/`, `/query/`, `/suggest` 等
- **第三方服务**: Google API、Facebook API 等

### 📊 数据展示
- 请求方法颜色标识（GET/POST/PUT/DELETE）
- 实时状态显示
- 响应时间统计
- 请求头分析
- 查询参数解析

## 安装方法

### 开发者模式安装
1. 下载或克隆此项目
2. 打开Chrome浏览器，进入 `chrome://extensions/`
3. 开启右上角的「开发者模式」
4. 点击「加载已解压的扩展程序」
5. 选择本项目文件夹

### 生成图标
由于图标文件需要手动生成，请按以下步骤操作：
1. 在浏览器中打开 `generate-icons.js`
2. 在控制台中运行 `generateAllIcons()`
3. 下载生成的PNG文件到 `icons/` 文件夹

## 使用说明

### 基本使用
1. 安装扩展后，访问任意网页
2. 点击浏览器工具栏中的扩展图标
3. 扩展会自动开始捕获网络请求
4. 在弹出窗口中查看捕获的请求

### 控制面板
- **启用/禁用捕获**: 控制是否捕获网络请求
- **清空数据**: 清除已捕获的请求
- **导出数据**: 将捕获的数据导出为JSON文件
- **查看详情**: 查看请求的详细信息

### 过滤和搜索
- **按类型过滤**: 点击过滤标签查看特定类型的接口
- **搜索**: 输入URL或接口类型进行搜索
- **排序**: 按时间排序，最新请求在前

### 请求详情
点击任意请求项可展开查看：
- 完整URL
- 请求路径
- 域名信息
- 查询参数
- 请求体内容
- 请求头分析
- 错误信息（如果有）

## 技术实现

### 文件结构
```
api-capture-extension/
├── manifest.json          # 扩展配置文件
├── background.js          # 后台脚本（网络请求拦截）
├── content.js            # 内容脚本（页面注入）
├── popup.html            # 弹出窗口界面
├── popup.js              # 弹出窗口逻辑
├── generate-icons.js     # 图标生成脚本
├── icons/                # 图标文件夹
│   ├── icon16.png
│   ├── icon48.png
│   └── icon128.png
└── README.md            # 说明文档
```

### 工作原理
1. **Manifest V3**: 使用最新的扩展开发标准
2. **Web Request API**: 拦截和分析网络请求
3. **Content Script**: 在页面上下文中运行，增强捕获能力
4. **Service Worker**: 后台处理和分析请求数据
5. **智能识别**: 基于URL模式、请求方法、请求头等特征识别接口类型

### 接口识别算法
- **URL模式匹配**: 使用正则表达式匹配常见API路径
- **请求方法分析**: 根据GET/POST/PUT/DELETE判断操作类型
- **请求头分析**: 识别Content-Type、Authorization等关键头信息
- **请求体解析**: 分析JSON、表单数据等请求体格式

## 开发说明

### 自定义接口识别规则
在 `background.js` 中的 `apiPatterns` 对象中添加自定义规则：

```javascript
const apiPatterns = {
  custom: [
    /\/my-api\//i,
    /\/custom-endpoint/i
  ]
};
```

### 添加新的识别维度
1. 在 `identifyApiType` 函数中添加新的识别逻辑
2. 更新 `apiPatterns` 对象
3. 修改UI显示逻辑

### 调试方法
1. 在扩展管理页面点击「检查视图」
2. 查看控制台输出
3. 使用 `console.log` 调试

## 常见问题

### Q: 为什么有些请求没有被捕获？
A: 可能的原因：
- 扩展被禁用
- 请求发生在扩展加载之前
- 某些跨域请求受浏览器安全策略限制

### Q: 如何提高识别准确率？
A: 
- 添加更多自定义规则到 `apiPatterns`
- 分析请求头信息
- 结合URL路径和请求方法判断

### Q: 数据存储在哪里？
A: 数据存储在浏览器内存中，关闭弹出窗口后会保留，但关闭浏览器标签页后会清除。
